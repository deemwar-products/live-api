# AGENTS.md

Conventions for AI coding agents (and humans) working in this repository.

> **Routing rule:** before answering or acting, check whether a more specific
> `AGENTS.md` (or `CLAUDE.md`) exists closer to the file in question. If it
> does, **read it first and follow it** — local rules override these.
>
> Currently registered local files:
> - `apps/ui/AGENTS.md` — frontend (Vite + React + Tailwind) conventions.

## What this repo is

Voice + chat AI customer support platform. Three interfaces in one app:
**Customer** (voice/chat widget), **Business** (org admin dashboard), **Platform** (super admin). See `docs/prd.md` and `docs/trd.md`.

---

## Work style

- **One thing at a time.** The user drives what to build next. Don't scaffold whole features, abstractions, or files unprompted. Wait for direction.
- **Match the request's scope.** If the user says "fix errors", fix only errors — don't add styling, mocks, or refactors. If the user says "basic template", stop at the template.
- **Ask before assuming** on anything that has multiple reasonable interpretations. One short clarifying question is cheaper than redoing work.
- **No over-engineering.** No premature abstractions, no "while I'm here" cleanup, no comments explaining what the code does.

---

## General rules

- **Secrets:** never commit `.env`, credentials, or API keys. Use environment variables.
- **Generated files:** never edit build output (`dist/`, `node_modules/`, lock files generated by tooling).
- **TypeScript strict mode is on.** Don't silence with `// @ts-ignore`. Fix the type.
- **Lint before commit** if a lint script exists.
